# 📧 XJTLU 邮件智能助手 - 项目完整介绍与文件说明

## 🌟 第一部分：整个项目的完整介绍 (Project Overview)

本项目是一个基于 Python 自动化工具 Playwright 和 大语言模型 (LLM) 搭建的**“西浦 (XJTLU) 邮箱自动总结助手”**。
它的主要目的是为了解决学生们每天面对海量校内英文通知、图文并茂的长邮件时，阅读效率低、容易遗漏关键信息（如作业截止时间、社团活动报名等）的痛点。

### 核心亮点与工作流程：
1. **绕过双卡认证 (MFA)，直达邮箱**
   针对学校极其烦人的 MFA 验证，项目完全抛弃了“输入密码”的传统思路，直接采用 Cookie 注入。将浏览器登录状态“录入”后台后，无论如何都能免密极速拉取到信件。
2. **多层 iframe 穿透与“净水器”级别的正文提取**
   Outlook Web App (OWA) 的网页结构异常复杂。项目内置了“智能滚动”、“动态 Frame 穿透”等强壮机制。不仅能在变化莫测的页面元素中准确定位目标邮件，还能将抓取到的各种 HTML 脚本、图片标签如同过滤器一般剔除（利用 BeautifulSoup4），最后洗出用于 AI 分析的最纯洁的邮件文本内容。
3. **并行的多线程大模型分析提速**
   常规抓取10封长邮件逐个发给大模型分析会非常慢。本项目在底层引入了高并发（Batch 并行）逻辑，将长文本平均切分，每次3封左右并发丢给大语言模型（如 GPT-4o / DeepSeek），最后再汇总一次生成总摘要。大大缩短了用户的等待时间。
4. **小白也能一键用的 Web 交互界面**
   项目一改早期命令行黑底白字的生硬体验，用 FastAPI 挂载了一个清爽的网页可视化界面。用户可以直接输入关键词进行搜“特定话题的邮件”或是直接搜“最新邮件”，甚至可以在界面的“弹窗齿轮”里无脑粘贴最新的 Cookie 和配置，整个提取的过程更是有着实时动态日志（“正在提取第2封...”、“正在整理汇报...”）反馈，体验非常现代。

### 采用技术栈总结：
- **核心后端服务**：FastAPI + Uvicorn + Python asyncio 异步循环系统。
- **浏览器自驾辅助**：Microsoft Playwright（驱动后台隐藏状态的 Edge 浏览器）。
- **文本清洗器**：BeautifulSoup4。
- **网页前端**：纯 HTML5 + 内部原生 JavaScript 接口轮询 + 内联 UI 样式（无需懂前端框架就能随便改配色字体）。

## 📂 第二部分：每一个文件的每一个作用 (File-by-File Explanation)

### 【根目录文件】

- `README.md`
  **作用：项目的超级“说明书门面”。**详细介绍了本助手解决的三大核心技术挑战、功能特性、安装各种依赖以及详尽的图文快速启动部署步骤。这是任何人下载项目后第一眼看到并需要通读的。

- `维护指南.txt`
  **作用：开发者/继任者的“维修保养避坑手册”。**指导系统日常怎么去维护，特别是重点说明了当 Cookie 突然过期、又或者微软偷偷改了 Outlook 的按钮样式（导致抓不到内容报错）时，该如何立刻定位去替换配置、应对突发状况。

- `run_app.bat`
  **作用：Windows 系统下的小白“跑腿小弟”。**双击这个脚本，它会全自动帮你激活隐藏的 Python 虚拟环境，并把后端的 API 服务器拉起来，顺便自动打开你电脑自带的默认浏览器进入到 `127.0.0.1:8001` 的后台，一气呵成。

- `.env` 与 `.env.example`
  **作用：项目的“保险柜”。**专门存放那些如果你发到网上会被盗刷破产的 API 密钥信息（如 OPENAI_API_KEY）。`.example` 是特意留空的展示给别人看的模板，防偷核心。

- `.gitignore`
  **作用：代码库的“海关黑名单”。**告知 Git 版本控制系统哪一些绝对不要提交到 GitHub（比如写着你自己隐私账号的 `config.json`，包含了超大体积的 Python 运行缓存 `__pycache__` 等），绝对的隐私守护神。另会忽略根目录 `cookies_mail*.txt` 等 Cookie 导出，避免误提交。

- `校友产品优先级计划.md`
  **作用：产品路线图备忘。**从“个人工具”到“校友在线产品”的分阶段功能优先级（P1 上手与信任、P2 留存、P3 传播），便于和需求对齐。

- `项目更新跟进表.md`
  **作用：相对基准提交的改动快照。**记录 `git diff --stat` 规模与未跟踪文件说明，方便写周报或同步组员。

### 【src - 后端代码目录】

- `src/main.py`
  **作用：整个自动化系统的“发动机底层核心”（命令行入口）。**
  这里面全是最硬核干活的代码逻辑。它包含了唤醒浏览器、点击搜索输入框、翻页爬取、判断是否含有特定日期的正则匹配表达式，包含用 Beautifulsoup 暴力刮削核心文字的算法，更是最终去对接 OpenAI 兼容的大模型生成摘要的封装地。可以说是项目的绝对骨架。日常更多通过 Web 使用时，由 `app.py` 调用其中能力。

- `src/app.py`
  **作用：架起连接大脑和页面的“超级前台”（HTTP 服务入口）。**
  由于 `main.py` 全是底层黑乎乎的函数，需要有个门面。基于 FastAPI 写的这份文件就是门面。它把原本零散的逻辑整理成了 HTTP 接口请求（比如定义 `/api/execute` 来执行爬取）。同时，因为使用 Playwright 进行模拟登录极耗内存，这个文件也维护了一个叫全局 `AppState` 的资源池，确保浏览器打开一次就候场，不用每次查邮件重开。并与登录会话、按用户配置协同工作。

- `src/auth_db.py`
  **作用：本地账号与“每人一份配置”的存储层。**使用 SQLite（默认 `src/user.db`，已在 `.gitignore` 中忽略）保存注册用户信息（密码经 bcrypt）、以及各用户独立的邮箱与 AI 配置，供 `app.py` 在登录后读写。

- `src/config.json`（原始会搭配一个 `config.example.json`）
  **作用：控制运作指令的“方向盘参数台”。**
  里面是一个大号的字典，存储了现在你想用什么大语言模型（GPT-4、DeepSeek、阿里等）、你的独门 API Key、登录 XJTLU 邮箱必备的千字以上的长串 Cookie、以及如果后期更新 OWA 就必须要动态修正的搜索框 CSS 选择器（Selectors）。系统的所有“认路找人”资料都在这里。

- `src/requirements.txt`
  **作用：搬家师傅手里的“家当清点清单”。**
  记录了运行这个系统必须连带捆绑安装的一系列第三方工具库（哪怕缺一个都跑不起来）。分享给同学使用时，他只要在黑屏通过 `pip install -r requirements.txt` 就能自动复制出和你完全一模一样的运行土壤。

### 【src/templates - 前端目录】

- `src/templates/index.html`
  **作用：登录后的主控制台（网页交互界面本体）。**
  虽然只有一个文件，但它把结构 (HTML)、样式美化 (CSS) 和复杂的人机动作 (JS) 糅在了一起。你看到的输入搜索词框、进度弹幕条、设置齿轮弹窗，都是在这个单页里渲染的。这部分重点利用了 JavaScript 的 Fetch 和轮询技术（异步发送执行命令并反复找服务器要提取进度结果），最后优雅地把大模型写的摘要呈现在你的面前，做到了前端与后端的彻底解耦。

- `src/templates/login.html`
  **作用：登录页。**提交邮箱与密码，由后端校验会话后进入主界面。

- `src/templates/register.html`
  **作用：注册页。**创建本地账号（邮箱唯一），首次用户可将历史单用户 `config.json` 迁移到该账号配置（行为以 `app.py` 为准）。
