使用指南

1. 配置 AI 服务

点击顶部「设置」，填写 AI Base URL 和模型名称。API Key 为可选（留空将自动使用系统托管的 AI 服务）。保存后立即生效。

"ai": { "base_url": "https://api.openai.com/v1", "api_key": "sk-... (可选)", "model": "gpt-4o-mini" }

2. 连接邮箱

推荐：在「连接校园邮箱」区域点击「打开登录窗口」，在本机 Edge 中完成登录后回到此页点「我已登录」。

高级：在 Chrome 或 Edge 安装 Cookie-Editor，登录邮箱后导出为 JSON，粘贴到「高级：粘贴 Cookie」文本框并保存。

如果会话失效，可重新走上述流程覆盖保存。

3. 使用方式

  • 关键词留空时，按列表读取最近若干封邮件。
  • 填写总结指令后，系统会搜索、提取正文并生成摘要。
  • 摘要结果与搜索历史保存在本机浏览器存储中（按账号区分）。
  • 需要扫描很多邮件时，展开「高级：深度扫描」按引导分两步操作。

4. 定时摘要（每日 / 每周）

定时摘要会在你设定的时间之后，自动按固定节奏排队执行一次「日常」拉取与摘要，无需每天手动点「生成邮件摘要」。

点击顶部「定时摘要」打开右侧面板进行设置；面板右上角「收起」可隐藏面板（已保存的任务仍有效）。

  1. 先连接邮箱：完成连接并点击「验证连接」或交互登录通过。若会话失效，定时任务会失败或跳过，需重新连接后再等下一周期。
  2. 配置 AI：与手动摘要相同，在「设置」中填写可用的模型接口（或使用托管服务）。
  3. 按上文进入「定时摘要」面板后，打开「开启定时自动摘要」，选择「每日」或「每周」。选「每周」时，再选在一周中的哪一天触发。
  4. 设定时间：时间为服务器本地时钟（与页面提示一致）。到达该时刻之后，系统才会入队；实际执行可能再晚几秒到数分钟（后台排队、或与正在进行的分析串行）。
  5. 关键词（可选）：留空表示按邮箱列表顺序取最近若干封；填写则与手动分析相同，先按关键词搜索再取前 N 封。这里的「每日」指每个周期跑一次，不是严格的「过去 24 小时内每一封都包含」。
  6. 总结指令与条数上限：与手动摘要类似，可说明更关注活动、作业或通知等；条数上限受日常模式限制（页面会标注）。
  7. 点击「保存定时设置」。保存成功后，若已开启功能，可看到「下次大约触发」的参考时间；也可随时点「刷新」更新历史列表。
  8. 历史摘要：右侧列表按时间倒序展示每次结果。点击一条可展开查看完整正文；状态为「成功」「失败」或「跳过」时，说明文字会写在记录里。

请注意：定时摘要固定走「日常」拉取，与主区域「高级：深度扫描」相互独立。多人同时使用本服务时，任务会串行执行，可能排队。若某一周期已因失败或跳过结束，该周期内通常不会再次自动重试，需等到下一个日/周周期。
